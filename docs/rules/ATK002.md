# ATK002 — No revoked/deprecated ATT&CK techniques

## What it checks
Cited techniques are not marked `revoked: true` or `x_mitre_deprecated: true`
in the bundled ATT&CK STIX data.

## Why
Revoked or deprecated techniques have superseding IDs; using them
suppresses correct coverage reporting and confuses analysts.

## Bad example
```yaml
tags: [attack.t1086]   # PowerShell — revoked, replaced by T1059.001
```

## Good example
```yaml
tags: [attack.t1059.001]
```

## How to fix
Replace the deprecated ID with its replacement (see the MITRE ATT&CK
website's banner on the technique page).
