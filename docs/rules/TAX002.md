# TAX002 — Field-name modifiers are spelled correctly per Sigma 2.1.0

## What it checks
Modifiers attached to field names (e.g. `Image|endswith`) are spelled
correctly and from the Sigma 2.1.0 modifier vocabulary.

## Why
A misspelled modifier (`|endwith`) is parsed as a literal field name,
silently breaking the rule.

## Bad example
```yaml
detection:
  selection: { Image|endwith: '\cmd.exe' }
```

## Good example
```yaml
detection:
  selection: { Image|endswith: '\cmd.exe' }
```

## How to fix
Consult the Sigma 2.1.0 modifier list (`contains`, `startswith`,
`endswith`, `re`, `cased`, `base64`, etc.).
