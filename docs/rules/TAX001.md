# TAX001 — All detection field names exist in the configured taxonomy

## What it checks
Every field name used inside `detection.*` selectors is present in the
configured taxonomy (default: `sigma`).

## Why
Unknown field names silently match nothing on most backends — the rule
becomes a no-op.

## Bad example
```yaml
detection:
  selection: { Imagee: x }   # typo
  condition: selection
```

## Good example
```yaml
detection:
  selection: { Image: x }
  condition: selection
```

## How to fix
Fix the typo, or — if intentional — extend the taxonomy via the
`taxonomy` config key.
