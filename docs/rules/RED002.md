# RED002 — Title or id collides with a SigmaHQ public rule

## What it checks
The rule's `title` or `id` matches one already present in the vendored
SigmaHQ corpus.

## Why
Colliding IDs break downstream deduplication and SIEM imports.
Colliding titles confuse analysts during triage.

## Bad example
```yaml
id: 11111111-1111-1111-1111-111111111111   # already used in SigmaHQ
title: PowerShell Encoded Command          # already used in SigmaHQ
```

## Good example
A freshly generated UUIDv4 and a title that includes an
organization-specific scope tag.

## How to fix
Regenerate the `id` (`python -c "import uuid; print(uuid.uuid4())"`)
and rename the title to disambiguate.
