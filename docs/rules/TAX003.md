# TAX003 — Prefer canonical field over known aliases

## What it checks
When a known alias of a canonical Sigma field is used, suggest the
canonical name (e.g. `CommandLine` over `Commandline`/`Command_Line`).

## Why
Consistent field naming improves shareability across organizations and
backends.

## Bad example
```yaml
detection:
  selection: { Commandline: foo }
```

## Good example
```yaml
detection:
  selection: { CommandLine: foo }
```

## How to fix
Rename the field to the canonical Sigma spelling.
