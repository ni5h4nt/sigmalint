# FP002 — Prefer modifiers over leading/trailing wildcards

## What it checks
Values that start or end with `*` should be written with the
appropriate `|startswith`/`|endswith`/`|contains` modifier instead.

## Why
Modifiers are clearer, faster on most backends, and avoid surprising
regex semantics.

## Bad example
```yaml
detection:
  selection: { Image: '*\cmd.exe' }
```

## Good example
```yaml
detection:
  selection: { Image|endswith: '\cmd.exe' }
```

## How to fix
Drop the wildcard and add the matching modifier.
