# ATK001 — Every attack.t#### tag resolves to a known technique

## What it checks
Each `attack.t####[.###]` tag in `tags` resolves to a technique in the
vendored ATT&CK STIX bundle.

## Why
Unknown technique IDs are typos or refer to retired identifiers; they
mislead downstream coverage tooling.

## Bad example
```yaml
tags: [attack.t9999]
```

## Good example
```yaml
tags: [attack.t1059.001]
```

## How to fix
Look up the technique on https://attack.mitre.org/ and replace with the
canonical ID. Run `sigmalint update-data` to refresh the bundle.
