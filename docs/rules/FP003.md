# FP003 — Noisy log source has no negated filter selector

## What it checks
For log sources known to be high-volume (e.g. `process_creation`,
`network_connection`), the rule must contain at least one negated
filter selector in `detection.condition`.

## How "negated filter" is determined

This rule walks the parsed condition AST (see
`core.condition.has_negated_selector`) and checks for any selector
that satisfies *both*:

1. its name matches `filter`, `filter_*`, or starts with `_` (the
   Sigma convention for exclusion-style selectors), and
2. it appears under a `not` at any nesting depth.

Negation is propagated through grouping. The grouped form
`condition: selection and not (filter1 or filter2)` is recognized —
the inner `filter1` and `filter2` count as negated even though the
`not` syntactically guards an `Or` node, not them directly. This is
the only way to make the rule's intent (a structural property of
filtering) decidable from a condition string; surface-level
substring matching would miss the grouped case.

## Why
Without a negated filter, rules on noisy sources flood SOC queues on
deployment.

## Bad example
```yaml
logsource: { category: process_creation, product: windows }
detection:
  selection: { Image|endswith: '\reg.exe' }
  condition: selection
```
`reg.exe` is one of the genuinely hard-to-filter Windows binaries —
legitimate registry edits use it constantly, and no single
allowlist selector reliably partitions noise from signal. FP003
firing here is the **right call**: a rule with this shape will flood
SOC queues, and the author should either narrow the selection
(specific arguments, parent process) or accept that the rule belongs
in a hunting tier, not in an alerting tier.

## Good example
```yaml
logsource: { category: process_creation, product: windows }
detection:
  selection:    { Image|endswith: '\reg.exe' }
  filter_admin: { User|startswith: 'NT AUTHORITY\' }
  condition: selection and not filter_admin
```

## How to fix
Add a `filter_*` selector and reference it with `and not` in the
condition.
