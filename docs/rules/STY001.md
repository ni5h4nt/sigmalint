# STY001 — Top-level keys are lowercase

## What it checks
Every top-level key in the rule is lowercase.

## Why
Consistent casing prevents subtle bugs where two YAML keys differ only
in case and one silently overrides the other.

## Bad example
```yaml
Title: T
Detection: { sel: { Image: x }, condition: sel }
```

## Good example
```yaml
title: T
detection: { sel: { Image: x }, condition: sel }
```

## How to fix
Lowercase the offending key.
