# FP004 — Hardcoded environment-specific literals

## What it checks
Detection values contain literals that look environment-specific:
private IP ranges, single user names, internal hostnames, etc.

## Why
Such rules will not transfer between organizations and accumulate
silent breakage during environment changes.

## Bad example
```yaml
detection:
  selection: { DestinationIp: 10.0.7.42, User: 'CORP\jsmith' }
```

## Good example
Move environment-specific values into a Sigma Filter file
(`filters/<env>.yml`) and keep the rule generic.

## How to fix
Parameterize the values via Sigma Filters or via a tuning layer
downstream of the detection engine.
