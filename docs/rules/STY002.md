# STY002 — File uses LF line endings and .yml extension

## What it checks
The file ends in `.yml` (not `.yaml`) and uses LF line endings (no
CRLF).

## Why
Consistency with the SigmaHQ corpus and reproducible diffs across
platforms.

## Bad example
A file named `rule.yaml` with CRLF endings.

## Good example
A file named `rule.yml` saved with LF endings.

## How to fix
Rename to `.yml`; convert with
`dos2unix path/to/file.yml` or set `core.autocrlf=false` in git.
