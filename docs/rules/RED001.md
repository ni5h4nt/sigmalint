# RED001 — Detection fingerprint near-duplicates an existing SigmaHQ rule

## What it checks
Computes a fingerprint over `(logsource, detection)` and compares with
the vendored SigmaHQ corpus. Reports matches with Jaccard >= 0.85.

## Why
Duplicating an upstream rule wastes review effort and increases
maintenance cost.

## Bad example
A locally authored rule that mirrors `proc_creation_win_*.yml` from
SigmaHQ with cosmetic changes only.

## Good example
A rule that adds a distinct selector, a different logsource, or a new
filter on top of the upstream baseline.

## How to fix
Either contribute back to SigmaHQ, or document the intentional
divergence and disable RED001 for that file.
